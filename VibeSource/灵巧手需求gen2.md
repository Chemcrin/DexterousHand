## 三、仿生手部结构定义（严格匹配人类解剖学）
### 1. 整体结构
机械灵巧手包含：**腕关节（Wrist） + 5根手指（拇指、食指、中指、无名指、小指）**，所有关节命名、自由度、连接关系1:1仿生人类手部。

### 2. 关节命名与层级（核心规范）
| 手指类型          | 关节层级（从手掌到指尖） | 英文缩写  | 中文名称   | 自由度        |
| ------------- | ------------ | ----- | ------ | ---------- |
| 四指（食/中/无名/小指） | 掌指关节         | MCP   | 掌指关节   | 2（屈伸+外展内收） |
|               | 近侧指间关节       | PIP   | 近侧指间关节 | 1（屈伸）      |
|               | 远侧指间关节       | DIP   | 远侧指间关节 | 1（屈伸）      |
| 拇指            | 掌指关节         | MCP   | 拇指掌指关节 | 2（屈伸+外展内收） |
|               | 指间关节         | IP    | 拇指指间关节 | 1（屈伸）      |
|               | 虎口关节         | CMC   |        | 3          |
| 公共基底          | 腕关节          | WRIST | 腕关节    | 3（固定基准点）   |

### 3. 坐标体系（右手坐标系）
- 原点：**腕关节中心**
- X轴：手掌横向（拇指→小指方向）
- Y轴：手掌纵向（腕部→指尖方向）
- Z轴：垂直手掌平面（屈伸方向）

### 4. 骨骼长度参数（仿生比例，代码内可调节）
统一使用NumPy数组存储，单位：自定义长度单位（mm）
1. 掌骨（WRIST → MCP）：拇指最短，四指依次递减
2. 近节指骨（MCP → PIP）
3. 中节指骨（PIP → DIP）
4. 远节指骨（DIP/IP → 指尖）

## 四、可实现动作列表（基础+扩展，覆盖灵巧性需求）
### 1. 基础要求动作（必实现）
1. **OK手势**：拇指+食指指尖相触，其余三指自然弯曲
2. **精准抓握**：拇指对掌，四指轻曲，模拟捏取小物体
3. **握拳**：所有手指完全屈曲，贴合掌心
4. **竖大拇指**：拇指伸直上翘，其余四指完全屈曲

### 2. 扩展动作（丰富灵巧性，必实现）
1. **摊开手掌（初始姿态）**：所有关节完全伸直，手掌展开
2. **数字1手势**：仅食指伸直，其余手指弯曲
3. **数字2手势**：食指+中指伸直，其余手指弯曲
4. **数字3手势**：食指+中指+无名指伸直，其余手指弯曲
5. **数字5手势**：全手掌展开
6. **竖小拇指**：小指伸直，其余手指弯曲
7. **手指依次屈伸**：食指→中指→无名指→小指依次弯曲/伸直
8. **拇指对掌运动**：拇指从外侧向掌心旋转触碰各指尖
9. **半握拳**：手指中度弯曲，非完全握拳
10. **挥手动作**：手腕关节转动，连带五指轻微晃动

## 五、仿真功能需求
### 1. 基础渲染功能
1. 初始化3D画布，设置视角、坐标轴范围、标题
2. 按仿生结构绘制**所有关节点**（不同关节可区分颜色：MCP/PIP/DIP/IP）
3. 按层级绘制**所有骨骼线**（连接相邻关节，形成完整手型）
4. 固定帧率实时更新，无卡顿

### 2. 运动控制功能
1. 所有动作基于**关节角度控制**（NumPy数组存储各关节角度）
2. 角度→3D坐标变换：使用**旋转矩阵/欧拉角**（纯NumPy实现）
3. 动作平滑过渡：关节角度线性插值，避免跳变
4. 支持**单动作执行**、**动作循环播放**、**动作序列切换**

### 3. 交互与展示功能
1. 3D视图可旋转、缩放、平移（Matplotlib原生交互）
2. 动作切换可通过按键/循环自动执行
3. 画布显示当前动作名称
