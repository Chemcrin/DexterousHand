# 机械灵巧手 3D 仿真代码框架设计

## 1. 文档目的

本文档用于指导后续生成基于 Python、NumPy、Matplotlib 3D 的机械灵巧手仿真项目代码。本文档基于前期讨论，对原需求中的部分不符合真实人手构型的内容进行了修正，目标是构建一套更接近真实人手、结构清晰、动作自然、便于扩展的仿生机械手运动学仿真框架。

本项目定位为运动学仿真，而不是物理接触仿真。系统只负责根据关节角度计算三维骨架坐标，并通过 Matplotlib 进行实时可视化，不涉及碰撞、力反馈、摩擦、接触求解或刚体动力学。

## 2. 总体技术约束

1. 编程语言：Python 3.12。
2. 数值计算：仅使用 NumPy。
3. 可视化：使用 Matplotlib 3D，主要接口为 `mpl_toolkits.mplot3d`。
4. 依赖范围：`numpy`、`matplotlib`。
5. 仿真方式：基于关节角度的正向运动学。
6. 动作切换：通过角度插值实现平滑过渡。
7. 输出形式：一个可直接运行的 Python 文件，后续可按需要拆分为多个模块。

## 3. 坐标系定义

全局坐标系采用右手坐标系。

1. 原点：`WRIST`，即手腕中心点。
2. `+X` 方向：从拇指侧指向小指侧。
3. `+Y` 方向：从手腕指向指尖。
4. `+Z` 方向：朝向掌心方向，手指屈曲时向 `+Z` 方向弯曲。

默认张手姿态中，五根手指处于同一平面，即所有关节点的初始 `z = 0`。四指伸直并相互平行，方向均沿 `+Y`。拇指在掌面内自然张开，与四指方向约成 `30` 度夹角。

## 4. 仿生构型修正

原需求中存在以下需要修正的点：

1. 原需求将腕部定义为固定基准点。修正后，腕部应具有可活动姿态，并通过一段小臂线段展示腕部运动。
2. 原需求中拇指仅包含 `MCP` 和 `IP`。修正后，拇指应包含真实关键关节 `CMC`，否则无法自然完成对掌、OK、精准捏取等动作。
3. 原需求中手指长度描述为“四指依次递减”。修正后，长度比例应更接近真实人手：中指最长，小指最短，食指和无名指长度接近。
4. 原需求中挥手动作描述为腕部固定。修正后，挥手应主要由腕部或前臂姿态驱动。

## 5. 骨架拓扑结构

### 5.1 根部与小臂

根部可视化链条如下：

```text
FOREARM_BASE -> WRIST
```

`FOREARM_BASE` 是固定参考点，用于显示小臂方向。`WRIST` 是整只手的运动学根节点。腕部姿态变化时，整个手掌和所有手指都应跟随 `WRIST` 变换。

### 5.2 四指结构

食指、中指、无名指、小指均采用如下结构：

```text
MCP -> PIP -> DIP -> TIP
```

每根四指包含 4 个点和 3 段骨骼：

1. `MCP`：掌指关节。
2. `PIP`：近侧指间关节。
3. `DIP`：远侧指间关节。
4. `TIP`：指尖端点。

四指不额外建立显式掌骨活动关节。`MCP` 的空间位置由掌面基座参数固定定义。

### 5.3 拇指结构

拇指采用如下结构：

```text
CMC -> MCP -> IP -> TIP
```

拇指包含 4 个点和 3 段骨骼：

1. `CMC`：腕掌关节，也可理解为虎口基底关节，是拇指对掌运动的核心。
2. `MCP`：拇指掌指关节。
3. `IP`：拇指指间关节。
4. `TIP`：拇指指尖端点。

拇指必须显式建模 `CMC`。仅使用 `MCP + IP` 无法自然表现真实人手的拇指张开、内折、轴向旋转和对掌动作。

## 6. 自由度定义

### 6.1 根部自由度

虽然严格解剖学中旋前旋后主要来自前臂，而不是狭义腕关节，但为了满足仿真展示和挥手动作需求，代码中将根部姿态统一定义为 3 个通道。

变量名建议：

```python
wrist_flex_ext
wrist_radial_ulnar
forearm_prono_supination
```

含义如下：

1. `wrist_flex_ext`：腕部屈伸，绕全局或局部 `X` 轴旋转。
2. `wrist_radial_ulnar`：腕部桡偏/尺偏，绕全局或局部 `Z` 轴旋转。
3. `forearm_prono_supination`：前臂旋前/旋后，绕小臂方向或局部 `Y` 轴旋转。

推荐角度范围：

```python
WRIST_LIMITS = {
    "wrist_flex_ext": (-60.0, 70.0),
    "wrist_radial_ulnar": (-20.0, 30.0),
    "forearm_prono_supination": (-80.0, 80.0),
}
```

### 6.2 四指自由度

每根四指包含 4 个自由度。

变量名建议：

```python
mcp_abd_add
mcp_flex
pip_flex
dip_flex
```

含义如下：

1. `mcp_abd_add`：掌指关节外展/内收，控制手指在掌面内小幅摆动。
2. `mcp_flex`：掌指关节屈伸，控制手指向掌心方向弯曲。
3. `pip_flex`：近侧指间关节屈伸，只能向掌心方向弯曲或伸直。
4. `dip_flex`：远侧指间关节屈伸，只能向掌心方向弯曲或伸直。

推荐角度范围：

```python
FOUR_FINGER_LIMITS = {
    "mcp_abd_add": (-15.0, 15.0),
    "mcp_flex": (0.0, 90.0),
    "pip_flex": (0.0, 110.0),
    "dip_flex": (0.0, 80.0),
}
```

四指除了 `MCP` 之外，其余关节不允许产生侧摆或扭转。`PIP` 与 `DIP` 只允许屈伸。

### 6.3 拇指自由度

拇指包含 6 个自由度。

变量名建议：

```python
cmc_abd_add
cmc_flex
cmc_axial_rot
mcp_abd_add
mcp_flex
ip_flex
```

含义如下：

1. `cmc_abd_add`：拇指 `CMC` 在掌面内张开/收拢，决定拇指与四指的初始夹角和张开程度。
2. `cmc_flex`：拇指 `CMC` 向掌心方向内折。
3. `cmc_axial_rot`：拇指掌骨轴向旋转，用于让拇指指腹朝向其他手指，是对掌动作的关键。
4. `mcp_abd_add`：拇指 `MCP` 小范围侧向调整。
5. `mcp_flex`：拇指 `MCP` 屈伸。
6. `ip_flex`：拇指 `IP` 屈伸。

推荐角度范围：

```python
THUMB_LIMITS = {
    "cmc_abd_add": (-20.0, 35.0),
    "cmc_flex": (0.0, 45.0),
    "cmc_axial_rot": (0.0, 60.0),
    "mcp_abd_add": (-15.0, 15.0),
    "mcp_flex": (0.0, 55.0),
    "ip_flex": (0.0, 80.0),
}
```

## 8. 正向运动学设计

### 8.1 旋转矩阵函数

需要实现基础旋转矩阵：

```python
def rot_x(angle_rad: float) -> np.ndarray:
    pass

def rot_y(angle_rad: float) -> np.ndarray:
    pass

def rot_z(angle_rad: float) -> np.ndarray:
    pass
```

输入为弧度，输出为 `3x3` NumPy 矩阵。

### 8.2 任意轴旋转

拇指轴向旋转建议实现任意轴旋转矩阵。

```python
def rot_axis(axis: np.ndarray, angle_rad: float) -> np.ndarray:
    pass
```

实现方式：使用 Rodrigues 旋转公式。

### 8.3 根部姿态变换

推荐函数：

```python
def get_root_transform(root_angles: dict) -> np.ndarray:
    pass
```

返回 `3x3` 根部旋转矩阵 `root_R`。

建议旋转顺序：

```text
forearm_prono_supination -> wrist_radial_ulnar -> wrist_flex_ext
```

代码中可写为：

```python
root_R = rot_y(prono) @ rot_z(radial) @ rot_x(flex)
```

具体左右乘顺序需在代码中保持一致，并通过张手、挥手等姿态进行视觉验证。

### 8.4 四指正向运动学

推荐函数：

```python
def compute_four_finger_joints(
    finger_name: str,
    finger_angles: dict,
    root_R: np.ndarray,
) -> dict:
    pass
```

输出格式：

```python
{
    "MCP": np.ndarray,
    "PIP": np.ndarray,
    "DIP": np.ndarray,
    "TIP": np.ndarray,
}
```

实现步骤：

1. 读取 `PALM_BASE_POINTS[finger_name]` 得到局部 `MCP` 位置。
2. 通过 `root_R` 将 `MCP` 转换到全局坐标。
3. 初始方向使用 `FINGER_BASE_DIRECTIONS[finger_name]`。
4. 先应用 `mcp_abd_add`，使手指在掌面内小幅外展/内收。
5. 再应用 `mcp_flex`，使近节指骨向掌心方向弯曲。
6. `PIP` 与 `DIP` 继续沿当前局部坐标系向掌心方向屈曲。
7. 每一段骨骼长度来自 `BONE_LENGTHS[finger_name]`。

四指推荐旋转逻辑：

```text
base_R = root_R @ rot_z(mcp_abd_add)
proximal_R = base_R @ rot_x(mcp_flex)
middle_R = proximal_R @ rot_x(pip_flex)
distal_R = middle_R @ rot_x(dip_flex)
```

骨段方向初始为局部 `+Y`：

```python
local_forward = np.array([0.0, 1.0, 0.0])
```

坐标计算示意：

```python
MCP = root_R @ PALM_BASE_POINTS[finger_name]
PIP = MCP + proximal_R @ local_forward * length_mcp_pip
DIP = PIP + middle_R @ local_forward * length_pip_dip
TIP = DIP + distal_R @ local_forward * length_dip_tip
```

### 8.5 拇指正向运动学

推荐函数：

```python
def compute_thumb_joints(
    thumb_angles: dict,
    root_R: np.ndarray,
) -> dict:
    pass
```

输出格式：

```python
{
    "CMC": np.ndarray,
    "MCP": np.ndarray,
    "IP": np.ndarray,
    "TIP": np.ndarray,
}
```

实现步骤：

1. 读取 `PALM_BASE_POINTS["thumb"]` 得到局部 `CMC` 位置。
2. 读取 `FINGER_BASE_DIRECTIONS["thumb"]` 得到默认拇指方向。
3. 应用 `cmc_abd_add` 控制拇指在掌面内张开/收拢。
4. 应用 `cmc_flex` 控制拇指向掌心方向内折。
5. 应用 `cmc_axial_rot` 沿拇指掌骨方向旋转，使指腹朝向四指。
6. 应用 `mcp_abd_add` 和 `mcp_flex` 控制拇指中段姿态。
7. 应用 `ip_flex` 控制拇指末端屈曲。

拇指推荐旋转逻辑：

```text
thumb_base_align_R = align local +Y to thumb default direction
cmc_R = root_R @ thumb_base_align_R
cmc_R = cmc_R @ rot_z(cmc_abd_add)
cmc_R = cmc_R @ rot_x(cmc_flex)
cmc_R = cmc_R @ rot_axis(current_thumb_axis, cmc_axial_rot)
mcp_R = cmc_R @ rot_z(mcp_abd_add) @ rot_x(mcp_flex)
ip_R = mcp_R @ rot_x(ip_flex)
```

其中 `current_thumb_axis` 可取 `cmc_R @ local_forward`。

拇指默认张开角也可以通过 `thumb_base_align_R = rot_z(-30°)` 近似实现。

## 10. 动作模块设计

### 10.1 动作函数原则

所有动作函数只返回角度字典，不直接计算坐标。

推荐函数格式：

```python
def pose_open() -> dict:
    pass

def pose_ok() -> dict:
    pass

def pose_precision_grasp() -> dict:
    pass

def pose_fist() -> dict:
    pass

def pose_thumb_up() -> dict:
    pass
```

动作函数应从 `neutral_angles()` 或 `pose_open()` 拷贝一份默认角度，再修改相关关节。

### 10.2 默认角度函数

推荐函数：

```python
def neutral_angles() -> dict:
    pass
```

作用：

1. 返回所有关节为默认张手姿态的角度。
2. 四指全部伸直。
3. 拇指相对四指自然张开 `30` 度，此夹角由默认方向或 `cmc_abd_add` 表达。
4. 腕部姿态为零。

### 10.3 必实现动作

#### 10.3.1 摊开手掌

函数名：

```python
def pose_open() -> dict:
    pass
```

实现方式：

1. 四指所有屈曲角为 `0`。
2. 四指 `mcp_abd_add` 可给极小展开角，例如食指 `-4`、中指 `0`、无名指 `2`、小指 `5`，也可全部为 `0` 以满足四指平行要求。
3. 拇指保持约 `30` 度自然张开。
4. 腕部姿态为 `0`。

#### 10.3.2 OK 手势

函数名：

```python
def pose_ok() -> dict:
    pass
```

实现方式：

## 18. 关键风险与规避方式

### 18.1 拇指方向反了

风险：拇指默认方向或 `CMC` 旋转顺序错误，导致拇指向食指方向错误内凹，或者与真实右手相反。

规避方式：

1. 默认张手时先只显示拇指，不叠加任何屈曲。
2. 检查拇指是否在掌面内向拇指侧自然张开。
3. 确认 `cmc_flex` 是向掌心方向折入。
4. 确认 `cmc_axial_rot` 是让指腹朝向四指，而不是让拇指背面朝向四指。

### 18.2 腕部和中指黏连

风险：四指 `MCP` 基座点定义过近，导致中指 `MCP` 与 `WRIST` 几乎重合。

规避方式：

1. `middle` 的 `MCP` 基座点应具有明显正 `Y` 偏移。
2. 建议 `middle` 的局部基座位置类似 `np.array([-6.0, 52.0, 0.0])`。
3. 绘制时可增加淡色掌面辅助线，帮助观察基座结构。

### 18.3 手指反向弯曲

风险：屈曲轴定义错误，导致手指向手背方向折。

规避方式：

1. 统一规定 `+Z` 为掌心方向。
2. 检查 `rot_x(positive_angle)` 是否使局部 `+Y` 朝 `+Z` 弯曲。
3. 如果方向相反，应在屈曲角处使用负号或调整 `rot_x` 约定。

### 18.4 动作过于僵硬

风险：所有手指同时以相同角度弯曲，动作机械感过强。

规避方式：

1. 不同手指使用略不同屈曲角。
2. 过渡时使用平滑插值。
3. 相位动作加入轻微延迟。
4. 拇指动作优先由 `CMC` 驱动，而不是只靠 `MCP/IP` 硬弯。

## 19. 后续生成代码时的核心原则

1. 动作函数只返回角度，不直接返回坐标。
2. 正向运动学只关心角度到坐标的转换，不关心动作语义。
3. 渲染函数只关心坐标显示，不修改角度或骨架参数。
4. 骨骼长度必须保持恒定。
5. 默认张手姿态必须先视觉正确，再继续实现复杂动作。
6. 拇指 `CMC` 是仿生真实性的关键，不能省略。
7. 腕部应可动，挥手动作应由腕部或前臂姿态驱动。
8. `OK` 动作只需视觉近似，暂不引入逆运动学。
9. 新增动作时只需新增 `pose_xxx()` 函数并注册到动作库。
10. 所有角度都应经过限位检查，避免不符合真实人手的姿态。
